## 2. 实施目标
- 将 D1-2 clean 数据统一映射到路段与网格维度。
- 将多源时间戳统一到 15 分钟窗口，形成可用于建模的标准明细。

## 3. 输入与输出
- 输入：
  - `data-pipeline/governance/output/clean/*.csv`
  - `data-pipeline/alignment/config/spatial_mapping.json`
- 输出：
  - `data-pipeline/alignment/output/aligned/aligned_records.csv`
  - `data-pipeline/alignment/output/alignment_report.json`

## 4. 实施规则

### 4.1 空间映射规则
- 通过 `spatial_mapping.json` 按 source 映射字段（zone_field）转换：
  - `segment_id`
  - `grid_id`
- 未命中映射记录计入 `unmapped_rows`。

### 4.2 时间对齐规则
- 时间字段统一解析为 ISO8601。
- 对齐粒度：15 分钟。
- 计算：
  - `window_start = floor(timestamp, 15min)`
  - `window_end = window_start + 15min`

### 4.3 统一输出结构
- `source, entity_id, timestamp, window_start, window_end, segment_id, grid_id, metric_name, metric_value`
