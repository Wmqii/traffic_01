# A1-1 基线模型训练与评估实施说明（v1.0）

## 1. 文档信息
- 文档编号：`A1-1`
- 版本：`v1.0`
- 日期：`2026-03-27`
- 责任 Agent：`Model-Agent`
- 阶段：`M2`

## 2. 实施目标
- 基于 `data-pipeline/features/output/feature_table.csv` 按 `window_start` 时间轴切分训练/验证节点，训练 ARIMA 与树模型以 `metric_sum` 作为回归目标。
- 产出 RMSE/MAE/MAPE 等评价指标，并在 `model/baseline/output` 下留存指标、预测与报告文件，作为后续模型对比与评审依据。
- 记录 xgboost 缺失时的退而用之方案，并在文档中明确提示该依赖状态。

## 3. 范围与边界
- 范围内：
  - 特征配置由 `data-pipeline/features/output/feature_table.csv` 提供。
  - 模型脚本位于 `model/baseline/scripts/train_baseline.py`。
  - 输出固定在 `model/baseline/output` 下的 `metrics.json`、`predictions.csv`、`model_report.md`。
- 范围外：
  - 不涉及新的特征工程或特征扩充。
  - 不扩展其他指标或模型，只关注当前两种基线。
  - 不负责发布、部署或上游特征源的变更。

## 4. 输出物
| 类型 | 路径 | 说明 |
| --- | --- | --- |
| 特征表 | `data-pipeline/features/output/feature_table.csv` | 供训练使用的时间窗口汇总特征。 |
| 脚本 | `model/baseline/scripts/train_baseline.py` | 读取特征、拆分集、训练 ARIMA/树模型并导出结果。 |
| 指标 | `model/baseline/output/metrics.json` | 包含数据切分、模型类型与 RMSE/MAE/MAPE。 |
| 预测 | `model/baseline/output/predictions.csv` | 树模型验证集预测与聚合 ARIMA 预测。 |
| 报告 | `model/baseline/output/model_report.md` | 记录模型设置、度量与依赖信息。 |

## 5. 执行流程
1. 进入项目根目录，确认 `data-pipeline/features/output/feature_table.csv` 可读。
2. 运行 `python model/baseline/scripts/train_baseline.py`（可选传入 `--features` 或 `--output-dir` 改路径）。
3. 脚本按默认 75% 训练 / 25% 验证拆分数据；ARIMA 以聚合 `metric_sum` 训练，树模型使用 one-hot 与 lag 特征。
4. 完成后在报告中记录输出，并在执行日志中补充原始命令与指标。

## 6. 异常处理策略
- 若 `xgboost` 不可用，脚本将自动回退至 `sklearn.GradientBoostingRegressor`，并在报告中说明依赖情况。
- Statsmodels 会对极短序列发出“无频率”与“观测不足”的警告，属于预期现象，可忽略。
- 若特征文件路径或输出目录不存在，则脚本会抛出错误并停止，需人工校正路径后重新运行。

## 7. 验收标准
- 训练脚本执行返回 `0`，产出 `metrics.json`, `predictions.csv`, `model_report.md`。
- `metrics.json` 中树模型与 ARIMA 均包含 RMSE/MAE/MAPE 值，且树模型已记录回退依赖状态。
- `predictions.csv` 含验证集三条树模型记录与两条 ARIMA 聚合记录（当前样本大小）。
- `model_report.md` 反映模型信息、指标与依赖说明。
