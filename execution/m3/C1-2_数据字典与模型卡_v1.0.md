- 文档编号：`C1-2`

## 3. 数据字典

### 3.1 原始接入层（JSON Schema）

#### `metro.json`
| 字段 | 类型 | 说明 |
|---|---|---|
| `train_id` | string | 地铁车次ID |
| `timestamp` | date-time | 采样时间 |
| `station` | string | 站点 |
| `line` | string | 线路 |
| `occupancy_pct` | number(0-100) | 车厢拥挤度 |

#### `taxi.json`
| 字段 | 类型 | 说明 |
|---|---|---|
| `ride_id` | string | 出租车订单ID |
| `timestamp` | date-time | 采样时间 |
| `pickup_zone` | string | 上车区域 |
| `dropoff_zone` | string | 下车区域 |
| `passenger_count` | integer(>=0) | 乘客数 |

#### `weather.json`
| 字段 | 类型 | 说明 |
|---|---|---|
| `station_id` | string | 气象站ID |
| `timestamp` | date-time | 采样时间 |
| `temperature_c` | number | 温度（摄氏） |
| `precip_mm` | number(>=0) | 降水量（毫米） |
| `wind_kph` | number(>=0) | 风速（km/h） |

#### `event.json`
| 字段 | 类型 | 说明 |
|---|---|---|
| `event_id` | string | 事件ID |
| `timestamp` | date-time | 事件时间 |
| `location` | string | 事件位置 |
| `category` | string | 事件类别 |
| `expected_attendance` | integer(>=0) | 预估人流规模 |

### 3.2 对齐层（`aligned_records.csv`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `source` | string | 数据源（event/metro/taxi/weather） |
| `entity_id` | string | 源实体主键 |
| `timestamp` | date-time | 原始时间 |
| `window_start` | date-time | 15分钟窗起始 |
| `window_end` | date-time | 15分钟窗结束 |
| `segment_id` | string | 路段ID（地图匹配后） |
| `grid_id` | string | 网格ID |
| `metric_name` | string | 指标名 |
| `metric_value` | number | 指标值 |

### 3.3 特征层（`feature_table.csv`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `source` | string | 数据源 |
| `segment_id` | string | 路段ID |
| `grid_id` | string | 网格ID |
| `window_start` | date-time | 时间窗起始 |
| `window_end` | date-time | 时间窗结束 |
| `metric_name` | string | 指标名称 |
| `metric_sum` | number | 窗口聚合总和 |
| `metric_avg` | number | 窗口聚合均值 |
| `metric_max` | number | 窗口聚合最大值 |
| `metric_count` | integer | 记录数 |
| `hour_of_day` | integer | 小时特征 |
| `day_of_week` | integer | 星期特征 |
| `is_weekend` | integer(0/1) | 是否周末 |
| `is_peak_hour` | integer(0/1) | 是否高峰 |
| `segment_numeric` | integer | 路段数字编码 |
| `grid_area` | string | 网格分区 |
| `event_attendance_window_sum` | number | 事件人流外部特征 |
| `weather_precip_window_avg` | number | 天气降水外部特征 |
| `lag1_metric_sum` | number/null | 一阶滞后特征 |

### 3.4 质量评分层（`quality_scoreboard.csv`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `source` | string | 数据源 |
| `total_rows/clean_rows/reject_rows` | integer | 规模与清洗结果 |
| `missing_count/duplicate_count/anomaly_count/delay_count` | integer | 治理异常统计 |
| `aligned_rows/feature_rows` | integer | 下游覆盖情况 |
| `*_score` | number(0-100) | 各维度评分 |
| `overall_score` | number(0-100) | 总分 |
| `grade` | string | 等级 |
| `status` | string | PASS/FAIL |

## 4. 模型卡（A1-2 LSTM/GRU）

### 4.1 模型基本信息
- 模型名称：`A1-2 Deep Models`
- 版本：`m3-a1-2-v1.0`
- 任务类型：路段拥堵强度回归预测（窗口级）
- 模型族：`LSTM`、`GRU`
- 训练脚本：`model/deep/scripts/train_deep_models.py`

### 4.2 输入与特征
- 输入表：`data-pipeline/features/output/feature_table.csv`
- 时间窗：`15` 分钟
- 序列长度：`3`
- 特征配置：`data-pipeline/features/config/feature_spec.json`
  - `peak_hours`: `[7,8,9,17,18,19]`
  - `lag_steps`: `[1]`
  - 外部特征：`expected_attendance`、`precip_mm`

### 4.3 训练与数据概况（本版）
- `raw_rows`: `12`
- `window_rows`: `6`
- `sample_count`: `3`
- `train_samples`: `2`
- `val_samples`: `1`
- `feature_count`: `8`
- 运行后端：`tensorflow.keras`（未回退）

### 4.4 评估指标（本版）
| 模型 | RMSE | MAE | MAPE(%) |
|---|---:|---:|---:|
| LSTM | 15579.0918 | 15579.0918 | 708.1405 |
| GRU | 15170.0938 | 15170.0938 | 689.5497 |

说明：当前样本量较小，MAPE 偏高，模型用于流程闭环验证与能力演示；上线前需扩充样本并重训。

### 4.5 输出产物
- `model/deep/output/metrics.json`
- `model/deep/output/predictions.csv`
- `model/deep/output/model_report.md`

### 4.6 已知限制与风险
- 训练样本规模小（3 个序列样本），泛化能力有限。
- 指标波动受事件源样本影响较大，需在 M4 扩展数据覆盖。
- 若 `tensorflow` 不可用会回退 `sklearn.MLPRegressor`，需在版本说明中标记。

### 4.7 监控与治理建议
- 每次模型更新需同步：
  - 产出新版 `metrics.json` 与 `model_report.md`
  - 更新本模型卡版本号
  - 通过 `T1-2/T1-3` 门禁后再发布
