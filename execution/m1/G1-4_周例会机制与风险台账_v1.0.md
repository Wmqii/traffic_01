## 2. 周例会机制

### 2.1 会议目标
- 同步阶段任务进展、质量门禁状态、风险与阻塞。
- 对跨 Agent 依赖进行决策，防止里程碑偏移。
- 固化行动项和截止日期，形成可追踪闭环。

### 2.2 固定节奏
- 会议时间：每周五 `16:00-17:00`（Asia/Shanghai）
- 会议类型：`M1~M5 周例会`
- 组织角色：
  - 主持：PM-Agent
  - 记录：Doc-Agent
  - 质量门禁汇报：Test-Agent
  - 发布与稳定性汇报：Ops-Agent

### 2.3 议程模板（60分钟）
1. 进度盘点（15 分钟）：各 Agent 汇报本周完成与下周计划  
2. 风险与阻塞（20 分钟）：按风险等级逐项评估  
3. 质量门禁（15 分钟）：测试通过项/失败项/待补测项  
4. 决议与行动项（10 分钟）：负责人、截止日期、升级路径  

### 2.4 输出物
- 周例会纪要（使用模板：`G1-4_周例会记录模板.md`）
- 更新后的风险台账（`G1-4_风险台账.csv`）
- 任务板状态变更记录（`execution/03_M1启动批次任务板.csv`）

## 3. 风险台账机制

### 3.1 风险评级规则
- 概率：`1(低)~5(高)`
- 影响：`1(低)~5(高)`
- 风险分值：`概率 * 影响`
- 风险等级：
  - `H`：>= 16
  - `M`：9-15
  - `L`：<= 8

### 3.2 处置策略
- `H` 级风险：24 小时内形成缓解方案，并进入每周跟踪。
- `M` 级风险：72 小时内明确责任人与计划。
- `L` 级风险：纳入观察清单，按周复核。

### 3.3 升级机制
- 若风险处于 `H` 且持续超过 1 周未下降，升级至项目管理例会。
- 若阻塞超过 24 小时未解除，触发临时决策会。
