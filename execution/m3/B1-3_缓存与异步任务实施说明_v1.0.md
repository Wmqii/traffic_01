## 2. 实施目标

- 增加异步任务提交与状态查询机制。

## 4. 方案设计

### 4.1 缓存层
- 缓存后端选择：
  - `TRAFFIC_CACHE_BACKEND=auto`：优先 Redis，失败回退内存。
  - `TRAFFIC_CACHE_BACKEND=redis`：尝试 Redis，失败后仍回退内存并记录原因。
  - `TRAFFIC_CACHE_BACKEND=memory`：仅使用内存 TTL。
- TTL 配置：
  - `TRAFFIC_CACHE_TTL_SECONDS`（默认 `300` 秒）。
- Redis 连接：
  - `TRAFFIC_REDIS_URL`（默认 `redis://127.0.0.1:6379/0`）。
- 命中统计：
  - 返回 `hits/misses/keys/backend/fallback_reason`。

### 4.2 异步任务
- 使用 `ThreadPoolExecutor` 构建任务执行器。
- 任务状态：`pending` → `running` → `completed|failed`。
- 支持提交任务与查询状态，保存 `created_at/started_at/finished_at/result/error`。
- 示例任务：刷新热力层缓存（`refresh_heat_cache`）。

### 4.3 接口改造
- `GET /api/v1/map/layers/heat`：
  - 先查缓存，未命中则读取 repository 并回填缓存。
- 新增接口：
  - `GET /api/v1/cache/stats`（admin）
  - `POST /api/v1/cache/refresh/heat`（analyst/admin）
  - `POST /api/v1/tasks/cache/refresh-heat`（analyst/admin）
  - `GET /api/v1/tasks/{task_id}`（analyst/admin）
  - `GET /api/v1/tasks?limit=20`（analyst/admin）

## 5. 实施结果
- Redis 不可用时自动回退内存缓存，服务可运行。
- 热力层接口缓存命中统计可观察。
- 异步任务链路可提交、可查询、可列举。
- OpenAPI 已更新并导出。
