# B1-2 认证鉴权（JWT + RBAC）实施说明（v1.0）

## 1. 文档信息
- 文档编号：`B1-2`
- 版本：`v1.0`
- 日期：`2026-03-27`
- 责任 Agent：`Backend-Agent`
- 阶段：`M2`

## 2. 实施目标
- 为核心 API 引入 JWT 认证与 RBAC 角色权限控制。
- 提供登录、当前用户查询、角色拦截能力。
- 保持本地可运行、可验证、可导出 OpenAPI。

## 3. 变更范围
- 新增：`backend/auth.py`
- 修改：`backend/app.py`、`backend/README.md`、`backend/export_openapi.py`
- 输出：`backend/openapi.json`

## 4. 设计说明

### 4.1 JWT 认证
- 登录接口：`POST /api/v1/auth/login`
- Token 算法：`HS256`
- 默认过期：60 分钟（可通过环境变量配置）
- Token 载荷：`sub`（用户名）、`role`、`iat`、`exp`

### 4.2 RBAC 角色
- 角色：`admin`、`analyst`、`viewer`
- 内置测试账号：
  - `admin/admin123`
  - `analyst/analyst123`
  - `viewer/viewer123`

### 4.3 接口权限矩阵
- `viewer/analyst/admin`：
  - `GET /api/v1/predictions/segments/{segment_id}`
  - `GET /api/v1/congestion/events`
  - `GET /api/v1/map/layers/heat`
- `analyst/admin`：
  - `GET /api/v1/attributions/events/{event_id}`
- `admin`：
  - `GET /api/v1/admin/audit`

## 5. 实施结果
- JWT 登录、鉴权依赖、角色拦截均已落地。
- `/api/v1/auth/me` 可返回当前登录用户信息。
- OpenAPI 已重新导出并包含认证接口。

## 6. 验收标准
- 登录成功可获取 token。
- 带 token 访问授权接口返回 200。
- 权限不足访问返回 403。
- OpenAPI 文件可导出。
