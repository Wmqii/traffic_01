## 5. 项目复盘

### 5.1 做得好的点
- 多 Agent 任务边界清晰，阶段门禁机制执行稳定。
- 关键能力均形成自动化脚本和证据产物，审计可追踪性高。
- 发布链路具备演练闭环（监控、灰度、回滚、应急）。

### 5.2 主要问题
- 个别 CSV 字段存在历史里程碑偏差（已在执行中修正）。
- Windows 控制台编码对中文路径存在偶发影响（已通过相对路径规避）。
- 文档与任务推进节奏需要持续并行，避免收口期集中堆积。

### 5.3 根因分析
- 任务长期迭代中，外部导入表（Jira/Trello）与内部任务板存在字段语义差异。
- 验证脚本最初未统一考虑多环境编码兼容。

### 5.4 改进行动（下一迭代）
1. 固化“任务板/Jira/Trello 字段对照表”并在每轮同步前自动校验。
2. 统一脚本输出规范，优先相对路径，避免终端编码异常导致非零退出。
3. 增加 M5 最终验收自动化聚合脚本（功能/性能/质量/安全/文档五类一键汇总）。
4. 在发布前增加一次全链路 rehearsal（含 UAT 抽样脚本）。
