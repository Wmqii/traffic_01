# F1-1 页面骨架实施说明 v1.0

## 1. 任务目标

- 交付可运行前端骨架，覆盖 `驾驶舱 / 实时 / 预测 / 归因 / 管理` 五个业务页面。
- 建立统一导航、筛选区、状态区与运行日志区，支持后续多 Agent 并行扩展。
- 完成基础 API 联调能力与降级策略，确保后端不可用时仍可演示。

## 2. 实施内容

1. 页面结构
- 新建 `frontend/index.html`，采用左侧控制面板 + 右侧业务主区布局。
- 提供五个主页面标签，切换后保持筛选条件与数据状态。

2. 交互与状态
- 新建 `frontend/assets/app.js`。
- 实现登录/退出、筛选（时间窗、路段、事件）、手动刷新。
- 支持本地缓存 `apiBase/token`（`localStorage`）。

3. 图表与可视化
- 新建 `frontend/assets/styles.css`，统一主题与响应式适配。
- 接入 ECharts 5，落地以下图表：
  - 拥堵热度趋势线图
  - 拥堵驱动雷达图
  - 预测分级漏斗图
  - 事件归因桑基图

4. 后端接口联调
- 登录：`POST /api/v1/auth/login`
- 预测：`GET /api/v1/predictions/segments/{segment_id}`
- 事件：`GET /api/v1/congestion/events`
- 归因：`GET /api/v1/attributions/events/{event_id}`
- 审计：`GET /api/v1/admin/audit`

5. 降级策略
- 任一接口失败（网络不可达、401/403/404）时，自动切换对应模块到 Mock/Fallback 数据。
- 在管理页运行日志中记录 `loaded from API` 或 `fallback` 结果，便于测试留痕。

## 3. 交付文件

- `frontend/index.html`
- `frontend/assets/styles.css`
- `frontend/assets/app.js`
- `frontend/README.md`

## 4. 风险与后续

- 当前前端为零依赖静态方案，适合 MVP 与联调阶段；后续可迁移到 Vue/React 组件化架构。
- 当前时间窗筛选为 UI 入口，后端暂未提供 time window 参数化查询，可在 B2 阶段补齐查询参数。
