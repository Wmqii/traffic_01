# D1-1 数据源接入实施说明（v1.0）

## 1. 文档信息
- 文档编号：`D1-1`
- 版本：`v1.0`
- 日期：`2026-03-27`
- 责任 Agent：`Data-Agent`
- 阶段：`M2`

## 2. 实施目标
- 完成 4 类公开交通相关数据源的最小接入骨架：`taxi/metro/weather/event`。
- 形成可重复执行的接入流程，输出统一 `staging` 数据与接入报告。
- 为后续 `D1-2（治理规则）`、`A1-1（基线模型）`提供标准化输入样本。

## 3. 范围与边界
- 范围内：
  - 数据源配置样例
  - 字段 Schema 定义
  - 示例数据文件
  - 接入执行脚本与结果报告
- 范围外：
  - 真实外部 API/消息队列接入
  - 生产数据库落库与调度编排

## 4. 目录与产物

| 类型 | 路径 | 说明 |
|---|---|---|
| 配置 | `data-pipeline/ingest/config/data_sources.sample.yaml` | 4类数据源声明 |
| Schema | `data-pipeline/ingest/schemas/*.json` | 字段类型与必填约束 |
| 样例数据 | `data-pipeline/ingest/samples/*.csv` | 每类3条示例记录 |
| 执行脚本 | `data-pipeline/ingest/scripts/ingest_runner.py` | 拷贝样例到 `staging` 并输出报告 |
| 执行结果 | `data-pipeline/ingest/staging/*.csv` | 标准化接入输出 |
| 接入报告 | `data-pipeline/ingest/staging/ingest_report.json` | 时间、行数、字段头信息 |

## 5. 字段口径（当前版本）

### 5.1 Taxi
- 字段：`ride_id,timestamp,pickup_zone,dropoff_zone,passenger_count`
- 约束：`passenger_count >= 0`

### 5.2 Metro
- 字段：`train_id,timestamp,station,line,occupancy_pct`
- 约束：`0 <= occupancy_pct <= 100`

### 5.3 Weather
- 字段：`station_id,timestamp,temperature_c,precip_mm,wind_kph`
- 约束：`precip_mm >= 0, wind_kph >= 0`

### 5.4 Event
- 字段：`event_id,timestamp,location,category,expected_attendance`
- 约束：`expected_attendance >= 0`

## 6. 执行流程
1. 读取 `samples/*.csv`
2. 拷贝到 `staging/*.csv`
3. 统计每个源的 `header` 与 `row_count`
4. 输出 `staging/ingest_report.json`

执行命令：
```powershell
cd d:\小桌面\交通拥堵\data-pipeline\ingest
python .\scripts\ingest_runner.py
```

## 7. 异常处理策略
- 缺少样例文件：脚本直接失败并退出（返回非 0）
- 样例目录为空：抛出 `No sample CSV files found under samples/.`
- 字段变更：先更新 `schemas/*.json` 再调整 `samples/*.csv`

## 8. 验收标准（D1-1）
- 4 类源均有配置、Schema、样例数据
- 接入脚本可执行且返回成功
- `ingest_report.json` 生成且 `total_samples = 4`
- 每类 `row_count >= 3`
