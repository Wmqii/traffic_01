## 3. 用户手册

### 3.1 登录与权限
- 登录接口：`POST /api/v1/auth/login`
- 默认账号：
  - `viewer / viewer123`（只读）
  - `analyst / analyst123`（分析）
  - `admin / admin123`（管理）
- 权限差异：
  - `viewer`：可查看预测、事件、地图图层。
  - `analyst`：可访问归因、模型健康、任务列表。
  - `admin`：可访问审计、模型发布与回滚能力。

### 3.2 访问入口
- 前端页面：`http://127.0.0.1:5500`
- 后端文档：`http://127.0.0.1:8000/api/v1/docs`
- OpenAPI：`http://127.0.0.1:8000/api/v1/openapi.json`

### 3.3 核心业务流程
1. 登录系统并进入驾驶舱。
2. 选择时间窗口和区域，查看拥堵事件与热力分布。
3. 打开预测与归因视图，定位高风险路段与主要驱动因子。
4. 在管理页执行导出（PNG/CSV/Markdown）并归档研判结果。

### 3.4 常用功能操作
- 预测查询：`/api/v1/predictions/segments/{segment_id}`
- 事件列表：`/api/v1/congestion/events`
- 事件归因：`/api/v1/attributions/events/{event_id}`
- 地图热力：`/api/v1/map/layers/heat`
- 导出中心：
  - 图表 PNG
  - 数据 CSV
  - 简报 Markdown

### 3.5 常见问题（用户侧）
- 问题：登录失败（401）
  - 处理：确认用户名密码；检查 token 是否过期。
- 问题：归因接口 403
  - 处理：确认当前角色为 `analyst` 或 `admin`。
- 问题：页面图层不刷新
  - 处理：先执行接口 smoke 校验，再检查后端是否启动。

## 4. 运维手册

### 4.1 环境与脚本
- 环境文件：`ops/environments/dev.env|test.env|prod.env`
- 启停脚本：
  - `ops/scripts/start_env.ps1`
  - `ops/scripts/stop_env.ps1`
  - `ops/scripts/deploy_env.ps1`
- 校验脚本：
  - `ops/scripts/smoke_api.ps1`
  - `ops/scripts/monitoring_check.ps1`
  - `ops/scripts/gray_release_drill.ps1`
  - `ops/scripts/o1_5_runbook_drill.ps1`

### 4.2 标准操作流程
1. 部署：
```powershell
.\ops\scripts\deploy_env.ps1 -EnvName dev -NoInstall
```
2. 验证：
```powershell
.\ops\scripts\smoke_api.ps1 -EnvName dev
.\ops\scripts\monitoring_check.ps1 -EnvName dev
```
3. 停止：
```powershell
.\ops\scripts\stop_env.ps1 -EnvName dev
```

### 4.3 发布与回滚
- 灰度演练：
```powershell
.\ops\scripts\gray_release_drill.ps1 -EnvName dev -NoInstall
```
- 一体化应急演练：
```powershell
.\ops\scripts\o1_5_runbook_drill.ps1 -EnvName dev -NoInstall
```
- 模型回滚（A1-6）：
  - 提交 `POST /api/v1/tasks/model/rollback`
  - 查询 `GET /api/v1/tasks/{task_id}`
  - 校验 `GET /api/v1/model/registry` 与 `GET /api/v1/model/health`

### 4.4 审计与错误处理
- 审计摘要：`GET /api/v1/admin/audit`
- 审计事件：`GET /api/v1/admin/audit/events?limit=20`
- 标准错误结构：
  - body：`error.code/error.message/error.request_id/error.detail`
  - headers：`X-Request-ID`、`X-Error-Code`
- 常见错误码：
  - `AUTH_INVALID_CREDENTIALS`
  - `AUTH_FORBIDDEN`
  - `TASK_NOT_FOUND`
  - `VALIDATION_ERROR`
  - `INTERNAL_ERROR`

### 4.5 值班与SLA
- 关键故障响应：
  - `P1`：5分钟内恢复核心链路（MTTR<=300s）
  - `P2`：30分钟内恢复降级能力
- 值班检查最小集：
  - health 状态
  - 监控告警 critical=0
  - smoke 通过
  - 审计日志持续写入
