## 3. 用例分层

### 3.1 功能层
- A1-2：训练脚本可执行、产物完整、指标落盘。
- B1-3：缓存回退有效、异步任务状态流转正确。
- F1-3：高级图表渲染与交互触发正常。
- F1-4：地图/图表/表格/筛选器联动闭环。
- F1-5：导出中心可触发 PNG/CSV/简报导出。

### 3.2 接口层
- 鉴权与角色矩阵：viewer/analyst/admin 权限控制。
- B1-3 新增接口：
  - `/api/v1/cache/stats`
  - `/api/v1/cache/refresh/heat`
  - `/api/v1/tasks/cache/refresh-heat`
  - `/api/v1/tasks/{task_id}`
  - `/api/v1/tasks`
- OpenAPI 一致性：接口可见、字段契约无破坏。

### 3.3 性能层
- 热力图层接口缓存命中后响应时间基线（本地基线场景）。
- 前端图表联动响应（点击到筛选更新）观测延迟。
- 异步任务查询吞吐与稳定性抽样验证。

### 3.4 安全层
- JWT 有效性、过期 token、空 token、角色越权。
- 管理接口最小权限验证（viewer 禁止访问 admin 接口）。
- 导出能力不泄露敏感字段（仅导出展示层字段）。

### 3.5 可用性层
- dev/test/prod 脚本可启动、可回收、可重试。
- 前端 fallback 在后端异常时仍可演示。
- 日志与证据链可追溯（worklog/testlog/releaselog + 证据文件）。

## 4. M4 准入门禁（M3 退出标准）
- M3 关键任务完成率 >= 95%（必须包含 A1-2/B1-3/F1-3/F1-4/F1-5）。
- 门禁测试全绿：
  - 功能、接口测试通过率 = 100%
  - 无 `High/Critical` 未关闭缺陷
- OpenAPI、模型产物、前端 smoke 证据齐全并可复现。
- 日志与评审记录完整，可审计。

## 5. 缺陷分级与处理 SLA
- `Critical`：阻断核心流程或数据错误；`2小时内响应，24小时内修复或回滚`
- `High`：关键能力不可用但有替代路径；`4小时内响应，48小时内修复`
- `Medium`：局部功能异常；`1工作日响应，5工作日内修复`
- `Low`：体验或文档问题；`2工作日响应，迭代内修复`

## 6. 执行节奏与角色分工
- 执行节奏：
  - 每日增量回归（核心接口 + 前端 smoke）
  - 每周阶段回归（全量 M3 范围）
  - 阶段评审前执行门禁总检查
- 角色分工：
  - Test-Agent：计划维护、用例执行、缺陷裁定、门禁结论
  - Backend/Frontend/Algo Agent：缺陷修复、回归配合、证据补齐
  - PM-Agent：优先级决策、阻塞升级、评审收口
