## 4. 监控方案

### 4.1 巡检维度
- 系统层：
  - runtime 文件存在性
  - backend/frontend 进程存活
  - backend 错误日志行数阈值
- 应用层：
  - `/health` 可用性与响应时延
  - 登录链路（viewer）
  - 地图热力接口可用性
- 模型层：
  - `model/deep/output/metrics.json` 存在性
  - 模型最佳 MAPE 阈值
  - metrics 文件新鲜度（小时）

### 4.2 告警规则
- 配置文件：`ops/monitoring/alert_rules.json`
- 支持阈值：
  - health 响应时延 warning/critical
  - model MAPE warning/critical
  - model 指标时效阈值
  - backend 错误日志行数 warning/critical

### 4.3 输出产物
- 巡检结果：`ops/artifacts/o1_3_monitor_<env>.json`
- ECharts 看板配置：`ops/artifacts/o1_3_monitor_dashboard_<env>.json`
