## 4. 设计说明

### 4.1 配置隔离
- 使用 `ops/environments/*.env` 管理环境差异：
  - `API_HOST/API_PORT`
  - `FRONTEND_PORT`
  - `TRAFFIC_JWT_SECRET`
  - `TRAFFIC_JWT_TTL_MINUTES`
  - `LOG_DIR`

### 4.2 部署脚本
- `start_env.ps1`：启动 backend（uvicorn）与 frontend（http.server），写入 `ops/runtime/<env>.pids.json`。
- `stop_env.ps1`：读取 pid 文件并停止进程（已修复 `PID` 变量冲突）。
- `deploy_env.ps1`：封装 `stop + start + smoke`。

### 4.3 验证脚本
- `smoke_api.ps1` 验证以下链路：
  - `GET /health`
  - `POST /api/v1/auth/login`（viewer/admin）
  - `GET /api/v1/map/layers/heat`（viewer）
  - `GET /api/v1/admin/audit`（viewer=403，admin=200）
- 验证产物落盘：`ops/artifacts/o1_2_smoke_<env>.json`。

### 4.4 容器化模板
- 提供 `ops/docker/docker-compose.{dev|test|prod}.yml`，用于后续容器化部署演进。

## 5. 实施结果
- 三环境均可完成 `deploy + smoke` 闭环。
- `dev/test/prod` 产物均达到 `6/6 pass`。
- 部署脚本与回滚步骤已写入 `ops/README.md`。

## 6. 验收标准
- 三环境部署命令可执行。
- smoke 结果全部通过。
- 失败可通过 `stop_env.ps1` 回收并重试。
