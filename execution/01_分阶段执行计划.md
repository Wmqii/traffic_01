## 分阶段执行计划
| 阶段 | 执行任务 | 责任Agent | 进入条件 | 退出条件 | 关键依赖 | 风险与应对 |
|---|---|---|---|---|---|---|
| M1（第1-2周） | - 完成《项目章程/计划/里程碑冻结》及《需求基线与系统设计冻结版》文档整理与审批<br>- 制定多 Agent 协作规范（任务模板、状态机、日报与升级机制）并发布<br>- 建立周例会机制与风险/依赖台账，明确交叉协作粒度<br>- 输出仓库结构与 CI/CD 基线、周边文档模板（项目计划书、需求基线、系统设计快照） | PM-Agent + Ops-Agent + Doc-Agent | - 项目启动会完成，Stakeholder 与资源角色已确认<br>- 初步需求与架构框架形成，具备编写计划书的基本材料 | - 计划书、需求/设计冻结版、协作规范、风险台账、CI 基线正式存档<br>- 多 Agent 任务边界已明晰，审批完成 | - Stakeholder 签批（需求、计划、设计）<br>- 初始代码仓库与 CI 具备可用骨架 | 风险：审批延迟或需求不齐；应对：设定双方向审阅窗口、提前收集缺口清单；风险：CI/仓库资源未就绪；应对：Ops 提前预留 runner/权限。 |
| M2（第3-5周） | - 构建数据接入与治理管道（D1-1~D1-5）并验证基础表结构与质量<br>- 完成基线预测模型（ARIMA/XGBoost）训练与评估（A1-1）并产出报告<br>- 交付核心 API（预测、拥堵、归因基础）与认证鉴权（B1-1/B1-2）<br>- 构建前端基础页面与 WebGIS 图层（F1-1/F1-2）<br>- 完成 Dev/Test/Prod 环境部署与 CI/CD 验证（O1-2） | Data-Agent + Algo-Agent + Backend-Agent + Frontend-Agent + Ops-Agent | - M1 产出正式签收（计划书、协作规范、CI 基线）<br>- 核心数据源接口/凭据到位<br>- 基础计算与存储环境可用 | - 数据接入治理管道运行稳定、质量指标达标<br>- 基线模型达到 RMSE/MAE 目标、评估报告完成<br>- 基础 API 与前端骨架可联调，开发/测试环境部署完成 | - 外部数据源授权与接口稳定<br>- 存储/计算/网络资源（数据库、消息、权限）<br>- OpenAPI/接口文档（来自 M1 文档） | 风险：数据延迟或质量问题；应对：设立指标监控+回退到模拟数据；风险：模型评估未达标；应对：设定可验证的基线调整周期；风险：环境部署延迟；应对：Ops 设定备份节点与部署校验。 |
| M3（第6-8周） | - 训练与部署深度模型（LSTM/GRU）与时空模型（STGNN），并上线在线推理服务（A1-2/A1-3）<br>- 构建拥堵识别规则与归因分析 API（A1-4/A1-5 + Backend）<br>- 完成高级图表组件（至少 6 类）及地图-图表-筛选器联动与导出能力（F1-3/F1-4/F1-5）<br>- 支撑自动化测试（接口 + 前端 E2E）与测试计划基线（T1-1~T1-3）<br>- 输出接口文档、数据字典与模型卡（C1-1/C1-2） | Algo-Agent + Backend-Agent + Frontend-Agent + Test-Agent + Doc-Agent | - M2 功能（数据治理+基线模型+核心 API + 前端骨架）联调完成<br>- 自动化测试环境（CI）与数据准备好<br>- 计算资源和监控（model training）持续可用 | - 高级模型、拥堵/归因 API 与推理服务稳定运行<br>- 高级可视化与交互功能可演示，导出机制完成<br>- 自动化测试集覆盖关键场景，接口/前端 E2E 可执行<br>- 接口文档与数据/模型卡更新完成 | - 算力（GPU/推理服务）、训练数据质量<br>- 接口/模块定义（来自 M2 核心 API）<br>- 测试框架与用例库 | 风险：模型更新引发质量回退；应对：引入验证阈值并串联模型版本管理；风险：前端性能瓶颈或联动失效；应对：采用分批发布+性能基线；风险：测试脚本不稳定；应对：先稳定测试基线再扩展。 |
| M4（第9-10周） | - 完成模型版本管理与回滚策略（A1-6）、缓存与异步调度接口（B1-3/B1-4）、审计日志规范（B1-5）<br>- 进行性能压测与优化、安全测试（T1-4/T1-5）、监控告警面板与应急手册（O1-3/O1-5）<br>- 组织灰度发布与回滚演练并记录（O1-4） | Backend-Agent + Algo-Agent + Test-Agent + Ops-Agent | - M3 高级功能与测试通过基本烟囱（高级模型、交互、自动化测试）<br>- 性能/安全测试数据准备完成 | - 性能 P95 < 2s、地图交互 < 1s（缓存命中）<br>- 安全测试（鉴权、审计、限流）通过<br>- 监控/告警机制与应急脚本在预演环境验证通过<br>- 灰度与回滚流程黏性完成 | - 压测/安全测试平台（可生成负载）<br>- 监控堆栈（日志、Tracing、告警）<br>- 回滚预案与版本清单 | 风险：大规模压测或安全测试中发现严重缺陷；应对：分阶段修复+设置回滚窗口；风险：监控/告警漏报；应对：开展双向校验与事故演练；风险：发布演练成绩不达标；应对：提前执行彩排并记录改进。 |
| M5（第11-12周） | - 输出 UAT 验收报告与发布说明、上线版本归档（C1-3 + PM-Agent Stories）<br>- 整理用户手册、运维手册、培训材料，并同步培训/支持计划（C1-3）<br>- 编撰项目复盘与下一阶段迭代建议（C1-4 + PM-Agent）<br>- 完成功能/性能/质量/安全/文档验收故事，确保每一类验收标准达标 | PM-Agent + Doc-Agent + Ops-Agent + 全体配合 | - M4 发布准备与监控/安全报告完成<br>- Stakeholder/UAT 团队可用窗口已确认 | - UAT 正式签收，验收报告对齐所有四大类（功能、性能、质量、安全、文档）<br>- 上线版本与发布说明下发<br>- 手册与培训材料交付使用<br>- 项目复盘记录、后续迭代清单完成 | - Stakeholder/UAT 团队可用性<br>- 最终发布构件（版本、文档）<br>- 培训/运维支持资源 | 风险：UAT 阻塞或新需求篡入；应对：设定 UAT 预留修复日并锁定 Scope；风险：文档/发布说明落后；应对：并行撰写+Doc-Agent 负责人签收；风险：未形成复盘；应对：固定 PM-Agent 复盘会议与行动项。 |
